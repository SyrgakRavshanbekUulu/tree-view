# React + TypeScript + Vite

## Install Node modules

npm i

## Run project

npm run dev

## Алгоритмическая сложность:

```c
Время выполнения:
Время выполнения алгоритма зависит от количества категорий и их уровня вложенности.
Поскольку мы проходимся по каждой категории только один раз, время выполнения будет O(n),
где n - общее количество категорий.

Пространственная сложность:
Пространственная сложность зависит от размера дерева категорий и глубины его вложенности.
Мы храним небольшое количество состояния для каждой категории
(например, флаг для открытия/закрытия подкатегорий),
поэтому пространственная сложность также будет O(n),
где n - общее количество категорий.

```

## Выбор использования рекурсивного подхода для отображения дерева категорий в компоненте CategoryItem имеет несколько преимуществ:

```
Простота и читаемость кода:
Рекурсивный подход позволяет нам избежать дублирования кода и написать более компактный и читаемый код.
Мы можем использовать тот же компонент CategoryItem для отображения как родительских,
так и дочерних категорий.

Гибкость и масштабируемость:
Рекурсивный подход позволяет обрабатывать деревья категорий любой глубины вложенности.
Это означает, что наше решение будет работать независимо от того,
сколько уровней вложенности имеет дерево категорий.

Эффективное управление состоянием:
Использование локального состояния в компоненте CategoryItem для
открытия/закрытия подкатегорий позволяет нам эффективно управлять интерактивностью компонентов
 и обновлять только необходимые части интерфейса.

Альтернативным решением может быть использование библиотеки для работы с деревьями,
такой как react-treebeard или react-virtualized-tree.
Эти библиотеки предоставляют готовые компоненты для отображения и
управления деревьями и могут быть более мощными и гибкими в некоторых случаях.
Однако, для данной задачи, использование рекурсивного подхода может быть более простым и легким в понимании.

В целом, использование рекурсивного подхода для отображения
дерева категорий в компоненте CategoryItem представляет собой простое и
эффективное решение, которое отвечает требованиям задачи.
```
